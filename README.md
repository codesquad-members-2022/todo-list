### 커밋 템플릿, 이슈 템플릿

1. 커밋 템플릿

```
################
# feat : 새로운 기능 추가
# fix : 버그 수정
# docs : 문서 수정
# test : 테스트 코드 추가
# refact : 코드 리팩토링
# style : 코드 의미에 영향을 주지 않는 변경사항
# chore : 빌드 부분 혹은 패키지 매니저 수정사항
################

Example) feat: ~~~~기능 작업 진행 중
         description: todo add ~~식으로 기능 작업
```

2. 이슈 템플릿 -> 개인적인 이슈 사항들은 템플릿이 아닌 README & Notion에 정리
